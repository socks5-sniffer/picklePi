# Curriculum Overview 🎓

picklePi ships with a structured 10-level curriculum. Each level introduces new hardware components and Python concepts, building seamlessly on everything taught before it.

## 🎛️ Levels & Projects

| Level | Topic | Project | Skills | Badge |
|-------|-------|---------|--------|-------|
| **1** | Digital Output Basics | Hello, RGB LED | `gpiozero LED`, `signal.pause()`, color mixing | GPIO Initiate |
| **2** | Digital Input | Button Controls LED | `Button`, pull-up/down resistors, event-driven logic | Input Investigator |
| **3** | PWM & Analog Concepts | PWM Brightness | `PWMLED`, duty cycle, `pulse()`, for loops | PWM Tamer |
| **4** | Sensors & Reactive Systems | Buzzer Tones | `TonalBuzzer`, `Tone`, frequencies, musical notes | Signal Wrangler |
| **5** | Displays & User Feedback | Motion / Light Sensor | `MotionSensor`, callbacks, `when_motion` hooks | Sensor Specialist |
| **6** | Multi-Module Systems | Tilt & Shake Detector | I2C protocol, external libraries, data interpretation | Motion Intelligence |
| **7** | Mini Capstone Projects | Multi-Module Alarm System | State machines, Threading Event, system integration | Mini Systems Architect |
| **8** | Analog Input & ADC | Twist to Control | MCP3008 SPI ADC, voltage division, SPI Protocol | Analog Alchemist |
| **9** | Distance Sensing | Sonar Proximity Alert | `DistanceSensor`, time-of-flight math | Sonar Scout |
| **10** | Full System Capstone | The Smart Guardian | DS18B20 1-Wire, Relay, fail-safe design | Systems Architect |

## 📚 What Each Project Teaches

Every project is split into standard, digestible pages to provide a comprehensive learning experience:
1. **Project Overview** - The "why" and "what" behind the build.
2. **Hardware Setup** - Step-by-step wiring instructions paired with strict safety warnings.
3. **The Code** - A complete, working, runnable Python script.
4. **Code Walkthrough** - Line-by-line breakdown and explanation.
5. **Concept Deep Dive** - A look under the hood at the physical hardware and operating system layers.
6. **Experiment Mode** - Guided challenges that encourage learners to tweak and break the code on purpose.
7. **Troubleshooting** - Common pitfalls, errors, and how to fix them.
